# Quick Start — .NET Build Repo

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| .NET SDK | 8.0+ | https://dotnet.microsoft.com/download |
| Docker | 24+ | https://docs.docker.com/get-docker/ |
| Access to ADO project | — | Provided by platform team |

---

## Step 1 — Clone and run locally

```bash
git clone https://dev.azure.com/{ado-org}/{ado-project}/_git/{app-name}
cd {app-name}
dotnet run
```

Open http://localhost:8080 — you should see `{"message":"Hello, World!"}`.
Open http://localhost:8080/health — you should see `{"status":"ok"}`.

---

## Step 2 — Run tests

```bash
dotnet test tests/
```

Coverage reports are generated by Coverlet and land in `coverage/`.

---

## Step 3 — Publish locally

```bash
dotnet publish -c Release -o ./publish
dotnet ./publish/app.dll
```

---

## Step 4 — Build and run the Docker image locally

```bash
docker build -t {app-name}:local .
docker run --rm -p 8080:8080 {app-name}:local
```

Test the containerized app:

```bash
curl http://localhost:8080/health
# {"status":"ok"}
```

The Dockerfile uses `dotnet publish -c Release` inside the SDK stage and copies the output into the ASP.NET runtime image. The runtime image (`mcr.microsoft.com/dotnet/aspnet:8.0`) has no SDK or build tools.

---

## Step 5 — Set the CONTAINER_IMAGE pipeline variable

1. In ADO, navigate to **Pipelines → {app-name} → Edit → Variables**
2. Add a variable named `CONTAINER_IMAGE` with value `myacr.azurecr.io/myorg/{app-name}`
3. Do not lock the variable (it is not a secret)

---

## Step 6 — Trigger the first pipeline run

Push a commit to `main` (or queue the pipeline manually in ADO).

Expected run sequence:

1. **Test stage** — `dotnet test` with Coverlet; VSTest + Cobertura results in the **Tests** tab
2. **Build stage** — Docker build + push
3. **image-meta artifact** — published under **Artifacts**

---

## Step 7 — Verify the config-repo trigger

After the build pipeline completes on `main`, check **Pipelines → {app-name}-k8s-manifests** — a new run should queue within ~60 seconds.

---

## Common first issues

| Symptom | Fix |
|---------|-----|
| `dotnet: command not found` | Install .NET 8 SDK and ensure `dotnet` is on `PATH` |
| App binds to wrong port locally | `launchSettings.json` may override the port; set `ASPNETCORE_URLS=http://+:8080` or check `Program.cs` |
| Docker `ENTRYPOINT ["dotnet", "app.dll"]` fails | `AssemblyName` in `HelloWorld.csproj` must be `app`; verify it matches |
| Coverage file not found in pipeline | The pipeline expects `coverage/cobertura.xml`; ensure Coverlet output path matches |
